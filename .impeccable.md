# ECHO-VIS

## Design Context

### Users
Technical presenters — students and engineers building animated protocol,
dataflow, and cryptography diagrams (nodes, links, tokens, monitors, graphs)
that they export as GIF/APNG/PPTX packages for slide decks. They work in long
focused editing sessions and already understand the domain (variables, tokens,
keyframes), so the UI should be precise, not hand-holdy.

### Brand Personality
Precise, calm, engineered. A dark professional editor (purple/blue accents on
near-black panels) closer to a video editor or IDE than a consumer tool.
Confidence comes from clarity and consistency, not decoration.

### Aesthetic Direction
Dark theme, compact density, small type, draw.io/Figma-style chrome.
Anti-references: consumer-app chattiness, helper paragraphs, emoji.

### Design Principles
1. **Sentence case everywhere** — buttons, menu items, labels, chips
   ("Add node", "Reset to auto"). Feature names like Resolve stay capitalized.
2. **No invented abbreviations** — "Label color", not "Lbl clr". Tight spaces
   earn a shorter *word*, never a truncation.
3. **Tooltips over paragraphs** — detailed guidance lives in hover hints on
   section headers and labels; the only always-visible hints are one-line
   empty states.
4. **One name per concept** — node, link, token, morph, joint, junction,
   divider, area. Never vary terms for the same thing.
5. **Counters and values are spelled out** — "3 nodes · 2 links", not "3n · 2l";
   dropdowns show human labels, never raw enum values.
